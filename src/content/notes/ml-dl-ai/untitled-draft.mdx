Traditional computer programs work by following instructions a human wrote: "if this happens, do that." This flow of information operates on the premise that the designer of the program has explicitly defined the rules for all calculations.

<figure><img alt="image.png" src="https://pub-f80a55ef0e254b3db56b34af9b904cfe.r2.dev/d3463a9d-9c10-4c19-856f-5b584478e895/bb0fd485-3ec7-4e16-9b3d-a4a9e15ad1a0.webp"><figcaption>Traditional Computer Programs</figcaption></figure>

Machine learning flips the approach. Instead of writing rules, you give a computer lots of examples and let an algorithm find the patterns itself. The computer essentially teaches itself the rules by studying the data.

<figure><img alt="image.png" src="https://pub-f80a55ef0e254b3db56b34af9b904cfe.r2.dev/d3463a9d-9c10-4c19-856f-5b584478e895/16ea6af9-0ca1-4f05-ad76-024adaeead03.webp"><figcaption>Machine Learning</figcaption></figure>

This shift matters because some problems are too complex or subtle to describe with explicit rules. Nobody can write down every feature that makes an email spam, but a machine learning model can learn to recognize the pattern from enough examples.

# Three main flavors

Machine learning generally falls into a few categories:

![image.png](https://pub-f80a55ef0e254b3db56b34af9b904cfe.r2.dev/d3463a9d-9c10-4c19-856f-5b584478e895/f0e0a059-a7e9-4db2-b197-49bd20fe579d.webp)

* **Supervised learning**: The model learns from **labeled examples** — inputs paired with correct answers — like predicting a house's price based on its size, location, and age.
* **Unsupervised learning**: The model looks for **hidden structure in data that has no labels**, such as grouping customers into segments based on shopping habits.
* **Reinforcement learning**: The model learns through **trial and error**, receiving rewards for good decisions, similar to how a program might learn to play chess by playing millions of games against itself.

# Where deep learning fits in

![image.png](https://pub-f80a55ef0e254b3db56b34af9b904cfe.r2.dev/d3463a9d-9c10-4c19-856f-5b584478e895/37add1ec-7316-494d-abe2-8a4e5730ef63.webp)

Deep learning isn't a separate field from machine learning; it's a subset of it. The systems behind today's most impressive AI, like image generators and chatbots, use a particular structure called a neural network: layers of simple mathematical units *loosely* inspired by neurons in the brain. "Deep" simply means many of these layers stacked on top of each other.

<figure><img alt="Neural networks – TikZ.net" src="https://tikz.net/wp-content/uploads/2021/12/neural_networks-002.png"><figcaption>a simple multi-layered neural network</figcaption></figure>

What sets deep learning apart from classical ML methods (like decision trees or linear regression) is that it learns features automatically. With many older ML approaches, a person has to decide in advance which details of the data actually matter. With deep learning, you can feed a network raw pixels, and it learns on its own to detect edges in the early layers, shapes in the middle layers, and whole objects by the final layers — no one has to tell it what to look for.

That automatic feature learning is powerful, but it comes at a cost: deep learning typically needs far more data and computing power than classical methods, and it's harder to explain *why* it made a particular decision. So: all deep learning is machine learning, but not all machine learning is deep learning.
